# Button

The `Button` component is a reusable UI element designed to trigger actions. It supports four visual variants, optional left and right icons, a loading state, and all standard HTML button attributes.

---

## Variants

The button comes with four predefined style variants that control its color scheme and interactive states:

| Variant    | Description                                    | Colors & Behavior                                                                                         |
|------------|------------------------------------------------|----------------------------------------------------------------------------------------------------------|
| `primary`  | Default main action style                       | Background uses `--entchen-primary`, text uses `--entchen-text`. Includes hover and disabled styles.      |
| `secondary`| Less prominent secondary action                 | Background uses `--entchen-secondary`, text uses `--entchen-text-secondary`. Includes hover/disabled styles.|
| `danger`   | Indicates destructive or critical actions       | Red background with white text; distinct hover and disabled states using Tailwind red classes.            |
| `warning`  | Used for cautionary or warning messages         | Yellow background with black text; includes appropriate hover and disabled styling in yellow tones.       |

---

## Props

| Prop       | Type                        | Required | Default    | Description                                                                                      |
|------------|-----------------------------|----------|------------|-------------------------------------------------------------------------------------------------|
| `title`    | string                      | Yes      | —          | The main label text displayed inside the button.                                               |
| `leftIcon` | React.FC                    | No       | —          | Optional icon component rendered to the left of the title (hidden when loading).                |
| `rightIcon`| React.FC                    | No       | —          | Optional icon component rendered to the right of the title (hidden when loading).               |
| `loading`  | boolean                     | No       | false      | When true, disables the button and shows a spinner icon instead of any other icons/text changes.|
| `variant`  | `"primary" \|"secondary" \|"danger" \|"warning"`  | No       | `"primary"`  | Determines the visual style variant of the button.                                             |
| Other standard HTML `<button>` props like `disabled`, `onClick`, etc., are supported via extension.|

---

## Accessibility

- The button disables itself automatically when either the native `disabled` prop or the custom `loading` prop is true.
- The spinner has appropriate ARIA attributes (`role="status"`, `aria-label="loading"`) to announce loading state.
- Uses semantic `<button>` element ensuring keyboard accessibility by default.

---

## Usage Example

```jsx
import { Button } from "entchenComponents";
import { ReactComponent as DownloadIcon } from "./icons/download.svg";

function Demo() {
  return (
    <>
      <Button title="Submit" onClick={() => alert("Submitted")} />

      <Button 
        title="Download"
        leftIcon={DownloadIcon}
        variant="secondary"
      />

      <Button 
        title="Delete"
        variant="danger"
        onClick={() => alert("Deleted")}
      />

      <Button 
        title="Warning"
        variant="warning"
        rightIcon={DownloadIcon}
        loading
      />
    </>
  );
}
```

---

## Styling Details

- Base styles include rounded corners, padding, inline-flex layout with gap between icon(s) and label.
- Each variant applies background color, text color, focus ring color, hover effects, cursor styles, and disabled states using Tailwind CSS classes or CSS variables.
- Loading state replaces icons/text with an animated SVG spinner using current text color for consistency.
