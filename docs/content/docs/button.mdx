# Button

The `Button` component is a reusable UI element designed for user actions. It supports multiple visual variants, optional left and right icons, a loading state, and inherits all standard HTML `<button>` attributes for maximum flexibility.



---

## Variants

The button comes with four predefined style variants. Each variant has unique styling for its default, hover, focus, and disabled states.

| Variant | Description | Colors & Behavior |
| :--- | :--- | :--- |
| `primary` | The default style for the most important actions. | Uses CSS variables (`--entchen-primary`) for a themeable background. |
| `secondary`| For less prominent, secondary actions. | Also uses CSS variables (`--entchen-secondary`) for its background color. |
| `danger` | For actions that are destructive or critical. 🔴 | Uses a solid red background with white text for high visibility. |
| `warning` | For actions that require user caution. 🟡 | Uses a solid yellow background with black text. |

---

## Props

In addition to the props below, this component accepts all standard attributes of an HTML `<button>` element (e.g., `onClick`, `type`, `name`).

| Prop | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `title` | `string` | Yes | — | The text label displayed inside the button. |
| `variant` | `"primary" \| "secondary" \| "danger" \| "warning"` | No | `"primary"` | Determines the visual style of the button. |
| `leftIcon` | `ReactNode` | No | — | An icon or element to render to the left of the title. Hidden when `loading`. |
| `rightIcon` | `ReactNode` | No | — | An icon or element to render to the right of the title. Hidden when `loading`. |
| `loading` | `boolean` | No | `false` | If `true`, the button is disabled and a spinner is shown in place of any icons. |
| `disabled` | `boolean` | No | `false` | Standard HTML disabled attribute. Disables the button and applies disabled styles. |

---

## Accessibility

* The component renders a semantic `<button>` element with a default `type="button"` to prevent accidental form submissions.
* It automatically becomes disabled when either the `disabled` or `loading` prop is `true`, setting both the `disabled` and `aria-disabled` attributes.
* When loading, the spinner has `role="status"` to announce the loading state to assistive technologies.

## Usage Example

```jsx
import { Button } from "entchen-components";
import { DownloadIcon, TrashIcon } from "your-icon-library";

function Demo() {
  return (
    <>
      <Button title="Submit" onClick={() => alert("Submitted")} />

      <Button
        title="Download"
        leftIcon={<DownloadIcon />}
        variant="secondary"
      />

      <Button
        title="Delete"
        variant="danger"
        leftIcon={<TrashIcon />}
      />

      <Button
        title="Processing..."
        variant="primary"
        loading={true}
      />
    </>
  );
}
```

## Styling Details

The `Button` is styled using **styled-components**, ensuring scoped and dynamic styling.

* **Base Styles**: The button has a base style that includes an `inline-flex` layout, center alignment, padding, rounded corners, and a bold font weight. It also features smooth transitions for `background-color` and `box-shadow`.
* **Variant Styles**: A dedicated function applies styles for each `variant`. This includes `background-color`, `color`, and unique states for `&:hover`, `&:focus` (which applies a `box-shadow` ring), and `&:disabled`.
* **Loading State**: When `loading` is `true`, an animated SVG `Spinner` is rendered. The spinner is created using CSS `keyframes` and inherits its color from the button's text color for a consistent look. The `leftIcon` and `rightIcon` are hidden during the loading state, but the `title` remains visible.
