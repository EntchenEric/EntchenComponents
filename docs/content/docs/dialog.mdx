# Dialog

The `Dialog` component is a fully accessible modal dialog that uses a render prop for its trigger, offering maximum flexibility. It supports animated transitions, customizable titles and descriptions, and configurable confirm/cancel buttons with callback handlers.



---

## Features

- **Flexible Trigger**: Uses a render prop for the trigger, allowing any button or element to open the dialog.
- **Smooth Animations**: Features animated backdrop and dialog panel with fade and scale transitions on open and close.
- **Fully Accessible**: Implements `role="dialog"`, `aria-modal`, `aria-labelledby`, and `aria-describedby` for screen reader support.
- **Keyboard Navigation**: Closes when the **Escape** key is pressed. Focus is managed within the dialog.
- **User-Friendly**: Closes when clicking the overlay outside the dialog content and locks page scroll when open to prevent background interaction.

---

## Props

| Prop | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `trigger` | `(open: () => void) => ReactNode` | Yes | ŌĆö | A render prop function that receives an `open` handler and must return a React node to act as the trigger. |
| `title` | `ReactNode` | Yes | ŌĆö | The main heading displayed inside the dialog. Supports strings or any React node. |
| `description` | `ReactNode` | No | ŌĆö | Optional descriptive text below the title. Supports strings or any React node. |
| `onConfirm` | `(close: () => void) => void` | No | ŌĆö | Callback fired when the confirm button is clicked. Receives a `close` function to dismiss the dialog. |
| `onCancel` | `(close: () => void) => void` | No | ŌĆö | Callback fired when the cancel button is clicked. Also receives a `close` function. |
| `onDialogClose`| `() => void` | No | ŌĆö | Callback fired just after the dialog's closing animation completes and it unmounts. |
| `confirmButton`| `ButtonProps` | No | `{ title: "Confirm" }` | Props passed to the confirm `<Button>` component. |
| `cancelButton` | `ButtonProps` | No | `{ title: "Cancel", variant: "danger" }` | Props passed to the cancel `<Button>` component. |

---

## Behavior & Accessibility

* **Opening**: The `trigger` function is called with an `openDialog` handler. When this handler is invoked, the dialog mounts and animates into view. Page scrolling is disabled.
* **Closing**: The dialog can be closed by pressing `Escape`, clicking the backdrop, or programmatically via the `closeDialog` function passed to the `onConfirm` and `onCancel` callbacks. Upon closing, the exit animation plays, and page scroll is restored.
* **ARIA Roles**: The component correctly wires up ARIA attributes to ensure the title and description are announced by assistive technologies.

## Usage Example

This example demonstrates how to use a standard `Button` as the trigger via the render prop pattern.

```jsx
import { Dialog, Button } from "entchenComponents";

function DeletePostExample() {
  const handleDelete = (closeDialog) => {
    console.log("Post deleted!");
    // Perform async delete operation...
    closeDialog();
  };

  return (
    <Dialog
      trigger={(open) => (
        <Button
          title="Delete Post"
          variant="danger"
          onClick={open}
        />
      )}
      title="Confirm Deletion"
      description="Are you sure you want to delete this post? This action cannot be undone."
      onConfirm={handleDelete}
      confirmButton={{ title: "Yes, Delete", variant: "danger" }}
      cancelButton={{ title: "No, Keep It", variant: "secondary" }}
    />
  );
}
```

## Styling Details

* The component is styled using **styled-components**.
* **Overlay**: A fixed-position backdrop covers the screen, animating its `background-color`, `opacity`, and `backdrop-filter` to create a blurred, semi-transparent effect.
* **Content Panel**: The dialog content itself animates in with a fade (`opacity`) and scale (`transform: scale()`) effect for a smooth entrance. It uses CSS variables like `--entchen-primary` for its background, ensuring theme consistency.
* **Animations**: All transitions are timed to `200ms` for a quick and responsive feel.
* **Buttons**: The action buttons are instances of the `Button` component, inheriting all its variants and styling capabilities.
